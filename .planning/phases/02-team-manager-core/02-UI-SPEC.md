## Spacing Scale

Inherited from Phase 1 — no changes. All values multiples of 4px.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `p-1` | Icon gaps, inline padding, drag handle indent |
| sm | 8px | `p-2` | Compact element spacing, option pill gap, badge gap |
| md | 16px | `p-4` | Card internal padding (all Phase 2 cards), page horizontal padding |
| lg | 24px | `p-6` | Section padding, drawer internal padding |
| xl | 32px | `p-8` | Layout gaps |
| 2xl | 48px | `p-12` | Major section breaks |
| 3xl | 64px | `p-16` | Page-level spacing |

**Phase 2 specific spacing:**

| Context | Value | Tailwind |
|---------|-------|----------|
| Gap between stacked page cards | 12px | `space-y-3` |
| Side-by-side lineup column gap | 12px | `gap-3` |
| Player row internal padding | 12px | `p-3` |
| Confidence badge internal padding | 4px horizontal, 2px vertical | `px-1 py-0.5` |
| Weather chip internal padding | 4px horizontal, 2px vertical | `px-1.5 py-0.5` |
| Waiver rank number column width | 32px | `w-8 shrink-0` |
| Drawer width (slide-in) | 400px max, full-width on mobile | `w-full max-w-[400px]` |
| TrendChart height | 120px | `h-[120px]` |
| Drag handle touch target | 44px wide | `min-w-[44px]` |

Exceptions: Drag handle column uses `w-6` (24px) visually but `min-w-[44px]` touch area via padding. Confidence score arc/badge: 28px diameter minimum to keep text readable.

**Badge/chip vertical padding exception:** `py-0.5` (2px) is used on all badge and chip components (confidence badge, weather chip, injury badge, swap badge, Sleeper badge, impact tag chips, season selector pills). This is an intentional exception to the 4px minimum — at `text-xs` (12px) font size, `py-1` (4px) creates visual bloat and makes inline badges misaligned within player rows. 2px vertical padding is approved for compact inline badges only; all other interactive and structural spacing remains on the 4px grid.

## Typography

Inherited from Phase 1 — 4 sizes, 2 weights. No new sizes or weights added.

| Role | Size | Weight | Line Height | Class | Phase 2 Usage |
|------|------|--------|-------------|-------|---------------|
| Page heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` | TeamPage section not used — cards use card headings |
| Card heading / section heading | 16px | 600 (semibold) | 1.2 | `text-base font-semibold` | Lineup card title, Waiver card title, Standings card title |
| Body / label / player name | 14px | 400 (regular) | 1.5 | `text-sm` | Player full name in player rows, drawer body copy, NL explanation paragraph |
| Player name (emphasized) | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold` | Player name in lineup row (bold for scanability) |
| Meta / badge / stat label | 12px | 600 (semibold) | 1.2 | `text-xs font-semibold` | Position badge, injury badge label, weather chip label, confidence badge, waiver rank, slot badge, swap badge, impact tag chips |
| Data values (numeric) | 12–14px | 400 | 1.2 | `font-mono text-xs` or `font-mono text-sm` | Projected points value, confidence number, FAAB bid range, weekly pts in chart tooltip |

**JetBrains Mono rule:** Apply `font-mono` to all numeric data values — projected points, confidence scores (the number itself), FAAB bid amounts, stat totals, chart axis tick labels, point deltas. Do NOT use mono for player names, position labels, or UI copy.

**Weights in use:** 400 (regular) and 600 (semibold) only. No font-medium, no font-bold, no font-light.

## Color

All tokens inherited from Phase 1. Phase 2 extends with semantic mapping for injury status and confidence tiers.

### Base Palette (unchanged from Phase 1)

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Background | `bg-bg` | `#0B0E14` | Page backgrounds, TeamPage shell |
| Surface | `bg-surface` | `#141822` | All Phase 2 cards, drawer background, bottom nav |
| Surface elevated | `bg-raised` | `#1B2030` | Player rows within cards, slot position badge, waiver rank cell |
| Border | `border-border` | `#262C3A` | All card/row borders default |
| Accent | `text-accent` / `bg-accent` | `#3DA9FC` | See accent reserved list below |
| Success | `text-success` / `bg-success` | `#5CC8A9` | High confidence indicator (≥70), positive point delta |
| Warning | `text-warning` | `#F2B66D` | Questionable/Doubtful injury status, medium confidence (55–69) |
| Danger | `text-danger` / `bg-danger` | `#F26D6D` | OUT/IR/Suspended/PUP injury badge, negative delta, "No strong call" banner border |
| Text primary | `text-text` | `#E8ECF1` | Player names, card body copy, drawer headings |
| Text secondary | `text-muted` | `#9AA3B2` | Meta info (team, position), placeholder, inactive labels |

**60/30/10 split for Phase 2:**
- 60% `bg-bg` — TeamPage shell, drawer overlay area
- 30% `bg-surface` / `bg-raised` — card bodies, player row backgrounds, drawer panel, nav bar
- 10% `bg-accent` — primary CTAs, swap badge, active mode toggle, focus rings, drag handle hover

**Accent reserved for:** "Swap suggested" badge background, waiver mode active toggle state, "Add to roster" CTA button (if shown), league switcher active league highlight border, drag handle icon on hover/active, focus ring on any interactive element, projected points value in the lineup optimizer card (the number that updates on drag).

**Injury status color mapping (semantic, not decorative):**

| Status | Color Token | Visual |
|--------|-------------|--------|
| Active / no designation | — | No badge shown |
| Questionable | `text-warning` / `bg-warning/15` | Q badge |
| Doubtful | `text-warning` / `bg-warning/20` | D badge |
| Out | `text-danger` / `bg-danger/15` | OUT badge |
| IR / Suspended / PUP / NA | `text-danger` / `bg-danger/10` | IR badge (muted) |

**Confidence score tier colors:**

| Tier | Range | Color |
|------|-------|-------|
| High | 70–100 | `text-success` |
| Medium | 55–69 | `text-warning` |
| Low | 0–54 | `text-danger` |

**Second semantic color:** `bg-danger` / `text-danger` — used only for OUT/IR injury badges, negative deltas, "No strong call" banner accent, and drop candidate labels. Never decorative.

## Screens and Interaction Contracts

### Surface A: TeamPage — Card Stack (DECISION-002, TM-01, TM-02, TM-04)

**Focal point:** The Lineup Optimizer card is the primary visual anchor — it is always card #1 and takes the most vertical real estate. The waiver wire card is subordinate.

**Layout:** Standard Layout shell (bottom nav visible). Scrollable vertical stack. Page header at top, then cards in order:
1. Lineup Optimizer Card (current vs optimal side-by-side)
2. Waiver Wire Card
3. Standings Card
4. Player Detail Drawer (overlay on top of stack — see Surface C)

**Page header:** `px-4 pt-10 pb-3`
- Team name: `text-base font-semibold text-text` (14px/600 is section heading — see Typography note: use `text-base` for the largest page element here since there is no 24px heading in Phase 2; Phase 2 does not have a standalone h1)
- Record + points + waiver rank: `text-xs text-muted` in a horizontal row with `·` separators; waiver rank number in `font-mono`
- League Switcher: positioned right side of page header row (see Surface E)

**Card shared structure:**
- Container: `bg-surface border border-border rounded-xl p-4`
- Card title row: `text-base font-semibold text-text` + optional right-side control (toggle, mode selector)
- Gap between cards: `space-y-3`
- Card border radius: `rounded-xl` (12px, matches Phase 1 pattern)

**Page loading state:** Skeleton — 3 stacked `animate-pulse bg-raised rounded-xl h-32` blocks matching card heights. No full-page spinner. No blank screen.

### Surface B: Lineup Optimizer Card (TM-01, TM-02, TM-15)

**Focal point:** The "Optimal" column is the primary scan target. The projected total delta line below the two columns is the secondary scan target.

**Layout:** Two-column flex grid inside the card.
- Column headers: "Current" (left) | "Optimal" (right) — `text-xs font-semibold text-muted uppercase tracking-wide`
- Column gap: `gap-3` (12px)
- Each column is `flex-1 min-w-0`
- Stacked player rows within each column — not a table. `space-y-1.5` between rows.

**Player row anatomy (extends existing PlayerCard):**
```
[drag-handle] [slot-badge] [name + meta] [points + confidence]
```

- Drag handle: `GripVertical` Lucide icon, `size={16}`, `text-muted`, left-aligned, hidden until row is hovered (`group-hover:opacity-100 opacity-0 transition-opacity`). Width 24px visual, 44px touch area via `px-2.5`.
- Slot badge: `w-8 h-8 rounded-lg bg-raised flex items-center justify-center text-xs font-bold text-muted` — same as Phase 1 stub.
- Player name: `text-sm font-semibold text-text truncate`
- Meta (position · team): `text-xs text-muted`
- Injury badge (see Surface F): inline after player name, `ml-1.5`
- Projected points: `text-sm font-semibold font-mono text-accent` — always shown, even if proxy value
- Confidence badge: small pill, right of points — `text-xs font-semibold font-mono` with color from confidence tier table above, `bg-{tier}/15 px-1 py-0.5 rounded`

**"Swap suggested" badge (TM-02):** Shown only on rows where current ≠ optimal assignment.
- Position: overlapping the right edge of the Current column player row, `absolute -right-1.5 top-1/2 -translate-y-1/2` relative to the row container.
- Style: `bg-accent text-white text-xs font-semibold px-1.5 py-0.5 rounded-full`
- Label: "SWAP"
- The corresponding Optimal column row gets `border-accent` border highlight: `border border-accent/50`

**Projected total row:** Below both columns.
- Current total | delta arrow | Optimal total
- Format: `{current_pts} → +{delta_pts}` or `{current_pts} → {optimal_pts}` if delta=0
- Numeric values: `font-mono text-sm font-semibold text-text`
- Delta: `text-success font-mono text-sm font-semibold` if positive, `text-danger` if negative

**Drag-to-override (TM-15):**
- Uses `@dnd-kit/sortable` with `PointerSensor` + `KeyboardSensor`
- Dragging a player row: source row gets `opacity-50 border-dashed border-accent`; drag ghost: semi-opaque clone of the row `bg-surface border border-accent shadow-lg`
- Drop target (valid slot): `border-accent bg-accent/10` highlight on the receiving row
- Drop target (invalid slot — position mismatch): `border-danger/50 bg-danger/10`
- After drop: row snaps into new position, projected total animates (150ms `transition-colors` on value change). Override indicator: small `text-xs text-muted italic` tag "(override)" appended to player name.
- Keyboard: `Enter` to pick up, arrow keys to move, `Enter` to drop, `Escape` to cancel. Announce slot change via `aria-live="polite"` region.

**Off-season state (no injury data):** Do not show injury badges. Show a muted inline note below the lineup card: `text-xs text-muted italic "Injury data available in-season."` (matches Pitfall 3 from RESEARCH.md).

### Surface C: Player Detail Drawer (TM-03, TM-08, TM-09, TM-11, TM-12, TM-13, TM-14)

**Trigger:** Tapping any player name/row in the lineup card or waiver card opens this drawer. Not a full-page nav — stays on TeamPage.

**Implementation:** Radix `@radix-ui/react-dialog` (already installed). Slide in from the right on desktop; slide up from the bottom on mobile (< 640px).

**Animation:**
- Desktop: slide in from right — `translate-x-full` → `translate-x-0`, 200ms `ease-out`
- Mobile: slide up from bottom — `translate-y-full` → `translate-y-0`, 200ms `ease-out`
- Overlay: `bg-black/60 backdrop-blur-sm` (matches Phase 1 modal overlay)

**Drawer container:**
- Desktop: `fixed inset-y-0 right-0 w-full max-w-[400px] bg-surface border-l border-border overflow-y-auto z-50`
- Mobile: `fixed inset-x-0 bottom-0 h-[85vh] bg-surface border-t border-border rounded-t-2xl overflow-y-auto z-50`
- Internal padding: `p-6`

**Drawer header:**
- Player name: `text-base font-semibold text-text`
- Position · Team · Bye week: `text-xs text-muted`
- Injury badge (if applicable): inline after name
- Close button: `X` Lucide icon, `size={20}`, `text-muted hover:text-text`, top-right corner, `aria-label="Close player details"`

**Drawer content sections (vertical stack, `space-y-5`):**

1. **Stat chips row** — horizontal scrollable row of `bg-raised rounded-lg px-3 py-2` chips:
   - Projected pts: label `text-xs text-muted "Proj Pts"` + value `font-mono text-sm font-semibold text-accent`
   - Confidence: label + value with tier color
   - Matchup grade: label + letter grade `text-sm font-semibold` (A/B/C/D scale, color: A=`text-success`, B=`text-text`, C=`text-warning`, D=`text-danger`)
   - Opp rank vs position: `text-xs text-muted "vs #N {POS}"` — number in `font-mono`

2. **Weather chip** (TM-09, shown only when applicable — outdoor stadium + ≥20 mph wind OR heavy rain/snow):
   - Chip: `bg-raised border border-border rounded-full flex items-center gap-1.5 px-2 py-1`
   - Icon: `Wind` / `CloudRain` / `CloudSnow` Lucide icon, `size={14}`, `text-warning`
   - Label: `text-xs font-semibold text-warning` — e.g., "23 mph wind" or "Heavy rain"
   - Below chip: `text-xs text-muted` — "Projection adjusted down"
   - Indoor stadiums: this entire section is absent (never render)

3. **Injury status section** (TM-04 — shown in-season only):
   - Full status label + last update time: `text-xs text-muted font-mono` for timestamp
   - Replacement suggestion (when OUT): `text-xs text-muted "Suggested: {player_name}"` — player name is tappable to open their drawer

4. **Usage trend row** (recent snap share + target share):
   - Two `bg-raised rounded-lg p-2` mini chips side by side
   - `text-xs text-muted` label + `font-mono text-sm font-semibold text-text` value

5. **"Positive game script" chip** (TM-11 — shown only when team favored by 10+ points, RB only):
   - Chip: `bg-success/10 border border-success/20 rounded-full px-2 py-1`
   - Label: `text-xs font-semibold text-success "Positive game script"`
   - Sub-label: `text-xs text-muted "Favored by {N} pts · expected workload up"`

6. **FAAB bid range** (TM-12 — shown only for FAAB leagues):
   - Label: `text-xs text-muted "Suggested FAAB bid"`
   - Value: `font-mono text-sm font-semibold text-text` — e.g., "$14 ± $3"
   - Range bar: `h-1 bg-raised rounded-full` with `bg-accent` fill at midpoint, fading to `bg-accent/30` at edges

7. **Trend chart** (TM-13):
   - Section heading: `text-xs font-semibold text-muted uppercase tracking-wide "This Season"`
   - Chart: Recharts `ResponsiveContainer width="100%" height={120}` containing `LineChart`
   - Axes: `XAxis` week number, `YAxis` hidden (no label, `domain={['auto','auto']}`), `margin={{ top: 4, right: 4, bottom: 4, left: -20 }}`
   - Line: `stroke="#3DA9FC"` (accent), `strokeWidth={2}`, `dot={false}`, `type="monotone"`
   - Tooltip: `bg-surface border border-border rounded-lg px-2 py-1.5 text-xs font-mono text-text` — value: "{N.N} pts wk {W}"
   - Axis ticks: `fontSize={10}` `fill="#9AA3B2"` (text-muted hex)
   - "vs this opponent" toggle: `text-xs font-semibold text-accent hover:underline cursor-pointer` to the right of section heading — switches data series; active state gets `underline`
   - Season selector (last 3 seasons): three pill buttons `bg-raised rounded-full px-2 py-0.5 text-xs font-semibold text-muted`; active season: `bg-accent/15 text-accent`

8. **Player news feed** (TM-14 — shown when news data is available):
   - Section heading: `text-xs font-semibold text-muted uppercase tracking-wide "Recent News"`
   - Each item: `border-l-2 border-border pl-3 py-1.5 space-y-0.5`
   - Source + timestamp: `text-xs text-muted` — timestamp in `font-mono`
   - Headline: `text-sm text-text`
   - Impact tag chip: `text-xs font-semibold rounded px-1 py-0.5` — injury=`bg-danger/15 text-danger`, usage=`bg-accent/15 text-accent`, coach=`bg-raised text-muted`
   - If TM-14 data source not yet resolved: section is hidden entirely (no empty state shown in drawer for this section)

9. **Natural-language explanation** (TM-03):
   - Section heading: `text-xs font-semibold text-muted uppercase tracking-wide "Analysis"`
   - Body: `text-sm text-text leading-relaxed` — one paragraph, template-generated (no LLM in Phase 2)
   - Loading state: 3-line skeleton `animate-pulse bg-raised rounded h-4 w-full`

**Drawer loading state:** Skeleton version of all sections — `animate-pulse bg-raised rounded` blocks matching section heights. Drawer opens immediately with skeleton; content populates as data arrives.

**Player comparison (TM-08):** Triggered by "Compare" button in drawer header (secondary ghost button `text-accent text-xs hover:underline`). Replaces drawer content with a two-column comparison view — same stat chips for both players side by side. "Winner" column gets `border border-success/30 bg-success/5` card wrapper. Three biggest factors listed as: `text-xs font-semibold text-text` factor name + `text-xs text-muted` one-line explanation. Close comparison: "Back to {player}" link.

### Surface I: "No Strong Call" Banner (TM-10)

**Trigger:** All flex options have confidence < 55.

**Placement:** Replaces the flex slot recommendation in the Optimal column. Does NOT replace the entire lineup card.

## Accessibility Baseline

All Phase 1 rules carry over. Phase 2 additions:

- dnd-kit keyboard navigation: fully operable without mouse (PointerSensor disabled when `prefers-reduced-motion` is active; keyboard-only mode still works via KeyboardSensor)
- Player detail drawer: focus trap via Radix Dialog. ESC closes. First focus target: player name heading.
- Add dialog: focus trap. First focus: FAAB bid input (FAAB leagues) or first drop candidate (rolling leagues). ESC closes without submitting.
- Confidence badges: do not rely on color alone — include the numeric value for screen reader users
- Weather chip: `aria-label="Weather alert: {full description}"` on icon; text description adjacent for non-visual access
- Injury badges: `title="{full status name}"` attribute + `aria-label="{status} — {player_name}"`
- League switcher: `role="combobox"` semantics via Radix DropdownMenu; keyboard: `Enter`/`Space` opens, arrows navigate, `Enter` selects, `Escape` closes
- "No strong call" banner: `role="alert"` so screen readers announce it immediately when rendered
- Drag-to-override: `aria-live="polite"` region with message "{player_name} moved to {slot_type}. Projected total: {N} pts."
- Touch targets: 44×44px minimum on all interactive elements — add button in waiver row, drawer close button, league switcher trigger
- Add button in waiver rows: `aria-label="Add {player_name} to roster"` on the `+` icon button
