**api.ts changes:**

1. Add `withCredentials: true` to the axios.create() config. This is required for the httpOnly refresh cookie to be sent on cross-origin requests.

2. Replace the existing 401 handler in `api.interceptors.response.use` with the refresh retry queue pattern from RESEARCH.md Pattern 7. The pattern:
   - On 401: if already retrying, queue the request behind the in-flight refresh
   - Set `_retry = true` on the original request to prevent infinite loops
   - Call `POST /auth/refresh` (cookie sent automatically via withCredentials)
   - On success: update Zustand store with new token; retry original request
   - On failure: clear auth store; redirect to `/login`
   - Always reset `isRefreshing = false` and `failedQueue = []` in finally block

3. Keep the existing request interceptor unchanged (attaches Bearer header from Zustand).

Full replacement pattern from PATTERNS.md:
```typescript
let isRefreshing = false
let failedQueue: Array<{ resolve: (v: string) => void; reject: (e: unknown) => void }> = []

api.interceptors.response.use(
  (res) => res,
  async (err) => {
    const originalRequest = err.config
    if (err.response?.status === 401 && !originalRequest._retry) {
      if (isRefreshing) {
        return new Promise((resolve, reject) => {
          failedQueue.push({ resolve, reject })
        }).then((token) => {
          originalRequest.headers.Authorization = `Bearer ${token}`
          return api(originalRequest)
        })
      }
      originalRequest._retry = true
      isRefreshing = true
      try {
        const { data } = await api.post('/auth/refresh')
        useAuthStore.getState().setAuth(data.access_token, data.user_id)
        failedQueue.forEach(({ resolve }) => resolve(data.access_token))
        originalRequest.headers.Authorization = `Bearer ${data.access_token}`
        return api(originalRequest)
      } catch {
        failedQueue.forEach(({ reject: rej }) => rej(err))
        useAuthStore.getState().clearAuth()
        window.location.href = '/login'
      } finally {
        isRefreshing = false
        failedQueue = []
      }
    }
    return Promise.reject(err)
  }
)
```

CRITICAL: Do NOT add `refresh_token` to the axios interceptor response handling in a way that stores it in state. The refresh token stays in the httpOnly cookie only — the interceptor only reads the new `access_token` from the response JSON body.

3. Add the `/auth/callback` route to handle the Google OAuth redirect. This route reads `?token=` and `?user_id=` from the URL and calls `setAuth`:
```tsx
import { useEffect } from 'react'
import { useSearchParams, useNavigate } from 'react-router-dom'

function AuthCallback() {
  const [params] = useSearchParams()
  const navigate = useNavigate()
  const setAuth = useAuthStore((s) => s.setAuth)

  useEffect(() => {
    const token = params.get('token')
    const userId = params.get('user_id')
    if (token && userId) {
      setAuth(token, userId)
      navigate('/connect', { replace: true })
    } else {
      navigate('/login', { replace: true })
    }
  }, [])

  return null
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /auth/callback?token= URL param | Access token briefly in URL; frontend must call setAuth immediately |
| httpOnly cookie → POST /auth/refresh | Cookie sent automatically; JS cannot read raw refresh token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | ?token= in URL | accept | 15-min access token; appears in browser history; frontend calls setAuth on mount and does not retain the URL. Replace with a one-time code in Phase 8 polish if needed. |
| T-08-02 | Spoofing | Refresh token interception | mitigate | httpOnly cookie; withCredentials sends it automatically; JS never reads raw token; SameSite=Lax prevents cross-origin sends |
| T-08-03 | Tampering | RequireLeague redirect | accept | hasLeagues in localStorage is a UX gate, not a security gate; backend always verifies membership independently |
</threat_model>
