<objective>
Extend ConnectPage to support Yahoo and ESPN alongside Sleeper. Yahoo uses a server-side OAuth redirect (no form needed — one click). ESPN private requires SWID + espn_s2 cookie paste with instructions; ESPN public requires only a league ID. The ?reconnect=yahoo query param (from the HealthBanner) opens ConnectPage in reconnect mode, skipping straight to the Yahoo button. Platform icons differentiate the three options visually.

Purpose: All three platforms are now connectable from a single unified UI. This completes the user-visible end of the Yahoo/ESPN connector.
Output: Extended `ConnectPage.tsx`, new `PlatformIcon.tsx`, minor `App.tsx` addition.
</objective>

| ESPN cookie form | SWID/espn_s2 submitted via POST body (JSON); validated server-side before storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11-01 | Information Disclosure | ESPN cookie fields in browser | mitigate | Input type="text" (not password) for SWID/espn_s2 because users paste from browser devtools; submitted over HTTPS; no logging of input values |
| T-03-11-02 | Spoofing | ?platform=yahoo query param | accept | Query param only affects UI state display (success card) — no server action triggered by this param |
</threat_model>
